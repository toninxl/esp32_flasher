⚡ ESP32 Flasher

Guia passo a passo para gravar firmware no seu ESP32 usando o executável.

📋 Pré-requisitos

1

Instalar o driver USB-Serial

O computador precisa reconhecer a placa ESP32 como porta serial. Instale o driver correspondente ao chip USB da sua placa:

  • CP210x (Silicon Labs) — Download
  • CH340/CH341 (WCH) — Download
  • USB nativo (ESP32-S2, S3, C3) — driver já incluso no Windows 10+
💡 Como saber qual driver? Verifique o chip pequeno próximo ao conector USB da placa. Geralmente está escrito "CP2102", "CH340G" ou similar.
2

Preparar os arquivos de firmware

Você precisa de até 4 arquivos .bin gerados pelo PlatformIO. Peça ao desenvolvedor ou exporte do projeto:

Arquivo Descrição Status
bootloader.bin Inicialização do chip Obrigatório
partitions.bin Tabela de partições da flash Obrigatório
boot_app0.bin Seletor de boot OTA Opcional
firmware.bin Sua aplicação (código principal) Obrigatório
⚠️ Importante Coloque todos os arquivos .bin em uma mesma pasta de fácil acesso, por exemplo C:\firmware\.

🔥 Gravando o Firmware

3

Conectar o ESP32 ao computador

Conecte a placa ESP32 ao computador via cabo USB. Aguarde o Windows reconhecer o dispositivo — você verá uma nova porta COM no Gerenciador de Dispositivos (ex: COM3, COM5).

💡 Verificar a porta Abra o Gerenciador de Dispositivos → Portas (COM e LPT) para ver qual porta foi atribuída.
4

Abrir o ESP32 Flasher

Dê duplo-clique no arquivo ESP32_Flasher.exe. A interface abrirá em modo escuro.

5

Configurar a conexão

Na parte superior da interface:

  • Porta: selecione a porta COM do seu ESP32 (ex: COM3). Clique em ⟳ para atualizar a lista.
  • Chip: selecione o modelo correto do seu ESP32:
    • ESP32 — placas DevKit V1, WROOM-32
    • ESP32-S2 — placas com S2-WROOM
    • ESP32-S3 — placas com S3-WROOM
    • ESP32-C3 — placas com C3-MINI
  • Baud: mantenha 921600 (padrão). Reduza para 460800 ou 115200 se tiver erros de conexão.
❌ Selecionar o chip errado pode impedir o boot! Os endereços de flash mudam entre chips. Verifique o modelo impresso na placa ou no módulo (ex: "ESP32-WROOM-32", "ESP32-S3-WROOM-1").
6

Selecionar os arquivos .bin

Clique em "Selecionar..." ao lado de cada linha para escolher os arquivos:

  • Bootloader: selecione bootloader.bin
  • Partitions: selecione partitions.bin
  • boot_app0: selecione boot_app0.bin (pode deixar vazio)
  • Firmware: selecione firmware.bin

Os endereços à esquerda (ex: 0x1000, 0x8000) são preenchidos automaticamente conforme o chip selecionado. Não altere os endereços a menos que saiba o que está fazendo.

7

Configurar opções (opcional)

  • ☐ Limpar flash antes de gravar — marque se quiser apagar toda a memória (útil para resolver problemas ou limpar configurações salvas no dispositivo).
  • ☑ Abrir monitor serial após gravar — recomendado. Permite ver as mensagens do ESP32 imediatamente após a gravação.
  • Baud monitor: mantenha 115200 (deve ser o mesmo configurado no código, em Serial.begin()).
8

Gravar!

Clique no botão "Gravar Firmware" (verde).

O progresso aparecerá na área de log. Você verá mensagens como:

  • Connecting.... — conectando ao chip
  • Writing at 0x00010000... (100%) — gravando
  • Hard resetting via RTS pin... — gravação concluída com sucesso!
✅ Sucesso! Ao ver "Hard resetting via RTS pin...", o firmware foi gravado. O ESP32 reiniciará automaticamente e o monitor serial exibirá a saída do seu programa.

📡 Monitor Serial

9

Usar o monitor serial

O monitor serial permite ver as mensagens que o ESP32 envia via Serial.println().

  • Automático: se a opção "Abrir monitor serial após gravar" estiver marcada, o monitor abre sozinho após o flash.
  • Manual: clique no botão "Monitor Serial" a qualquer momento para abrir/fechar.
  • Para fechar, clique em "Parar Monitor".
💡 Baud rate do monitor O baud rate do monitor deve ser igual ao configurado no código do ESP32 (Serial.begin(115200)). Se os caracteres aparecerem como "lixo", provavelmente o baud está errado.

📍 Endereços de Flash por Chip

Estes endereços são preenchidos automaticamente. Apenas para referência:

Chip Bootloader Partitions boot_app0 Firmware
ESP32 0x1000 0x8000 0xe000 0x10000
ESP32-S2 0x1000 0x8000 0xe000 0x10000
ESP32-S3 0x0 0x8000 0xe000 0x10000
ESP32-C3 0x0 0x8000 0xe000 0x10000

🔧 Problemas Comuns

A porta COM não aparece na lista
  • Verifique se o cabo USB está conectado corretamente.
  • Instale o driver USB-Serial correto (CP210x ou CH340).
  • Tente outro cabo USB — alguns cabos só carregam e não transmitem dados.
  • Clique no botão ⟳ para atualizar a lista de portas.
"Failed to connect" ou "Timed out waiting for packet header"

O ESP32 não conseguiu entrar no modo de bootloader. Tente:

  • Segurar o botão BOOT da placa, pressionar EN/RESET, e soltar o BOOT.
  • Depois, clique em "Gravar Firmware" novamente.
  • Algumas placas fazem isso automaticamente — se não funcionar, o circuito de auto-reset pode estar com problema.
"Permission denied" ou "Porta em uso"
  • Feche qualquer outro programa que esteja usando a porta serial (Arduino IDE, PlatformIO, PuTTY, outro terminal).
  • Se o monitor serial estiver aberto, ele será fechado automaticamente antes da gravação.
Monitor serial mostra caracteres estranhos ("lixo")

O baud rate do monitor está diferente do configurado no firmware.

  • Verifique no código do ESP32 qual valor está em Serial.begin().
  • Ajuste o campo "Baud monitor" na interface para o mesmo valor (geralmente 115200).
ESP32 não inicia após gravação / tela de boot em loop
  • Verifique se selecionou o chip correto na interface (ESP32, S2, S3 ou C3).
  • Chips errados = endereço do bootloader errado = o dispositivo não inicia.
  • Tente marcar "Limpar flash antes de gravar" e gravar novamente.
  • Verifique se os arquivos .bin são da versão correta do firmware.
Gravação muito lenta
  • Verifique se o baud rate de flash está em 921600.
  • Se estiver tendo erros com 921600, reduza para 460800.
  • Use um cabo USB curto e de boa qualidade.
"[Monitor] Dispositivo desconectado"
  • O cabo USB foi desconectado ou o ESP32 perdeu energia.
  • Reconecte o cabo e clique em "Monitor Serial" novamente.

🗑️ Limpar Flash

O botão "Limpar Flash" apaga toda a memória flash do ESP32, incluindo:

  • Firmware e bootloader
  • Configurações salvas (NVS, Wi-Fi credentials)
  • Arquivos no SPIFFS/LittleFS
⚠️ Use com cuidado Após limpar a flash, será necessário gravar todos os binários novamente para o ESP32 funcionar.
